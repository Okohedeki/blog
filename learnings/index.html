Kernel development

A practical, staged path for learning GPU / CUDA kernel development — built as a curriculum with deliverables, not a link dump. The spine: CUDA fundamentals → memory & performance → PyTorch integration → Triton → profiling → AI kernels → a portfolio capstone. LeetCode runs alongside so interview fluency builds without hijacking the plan.

Working principle GPU Mode is not a linear course — it's a resource graph (CUDA, Triton, torch.compile, papers, books, lectures, profilers, PyTorch internals, hardware docs). So I keep one spine and pull GPU Mode links in only when they support the project I'm on. The win is artifacts, not a collection of bookmarks.

B. The skill tree

Eight levels, bottom-up. Don't jump ahead — each level is the foundation for the next.

  1. CUDA syntax & execution model vector add, memory copy, indexing
  2. Parallel patterns reduce, scan, histogram, atomics
  3. Memory & performance coalescing, shared memory, tiling, occupancy
  4. Matmul & AI primitives matmul, softmax, layernorm, attention
  5. PyTorch integration profiler, memory snapshots, torch.compile, custom ops
  6. Triton softmax, matmul, layernorm, attention
  7. Real AI systems FlashAttention, quantization, KV cache, inference bottlenecks
  8. Portfolio Tiny Transformer Kernel Lab or GPU Order-Book Simulator

H. First 30 days

  • Week 1 — intro CUDA material; build vector add + saxpy; write a timing harness; LeetCode: arrays + hash maps (5).
  • Week 2 — thread/block/grid + memory hierarchy; build image blur / simple stencil; benchmark block sizes; LeetCode: two pointers + sliding window (5).
  • Week 3 — build reduction; compare global-memory vs shared-memory; profile with Nsight Compute; LeetCode: prefix sums + stack (4).
  • Week 4 — build scan/prefix sum; read GPU Gems scan / CUB conceptually; one-pager on scan & why it matters for state-space models; LeetCode: binary search + intervals (4).
