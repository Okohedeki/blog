Kernel development

Me teaching myself GPU / CUDA kernel work, tracked as a checklist so the progress is honest. The spine: CUDA fundamentals → memory & performance → PyTorch integration → Triton → profiling → AI kernels → a portfolio capstone. LeetCode runs alongside.

How I'm treating this GPU Mode isn't a linear course — it's a resource graph (CUDA, Triton, torch.compile, papers, books, lectures, profilers). So I keep one spine and pull links in only when they support the thing I'm building. The win is artifacts, not bookmarks.

B. The skill tree

Eight levels, bottom-up. Each one is the foundation for the next.

  1. CUDA syntax & execution model vector add, memory copy, indexing
  2. Parallel patterns reduce, scan, histogram, atomics
  3. Memory & performance coalescing, shared memory, tiling, occupancy
  4. Matmul & AI primitives matmul, softmax, layernorm, attention
  5. PyTorch integration profiler, memory snapshots, torch.compile, custom ops
  6. Triton softmax, matmul, layernorm, attention
  7. Real AI systems FlashAttention, quantization, KV cache, inference bottlenecks
  8. Portfolio Tiny Transformer Kernel Lab or GPU Order-Book Simulator
